# AstrBot Plugin Portrayal Core

_✨ 深度群友性格侧写 & 高清画像生成 (Core) ✨_

## 💡 简介

**人物画像(Core)** 是一款基于 AstrBot 的高级性格分析插件。

它不仅仅是简单的总结，而是通过深度回溯 OneBot 历史消息，利用大语言模型（LLM）进行心理学维度的侧写分析，并最终渲染为一张**极简风格的高清毛玻璃卡片**。

**🔥 核心特性：**
- **🎨 新的 UI**：基于 Playwright 的无头浏览器渲染，生成 Apple 风格（清透白昼模式）的毛玻璃卡片。
- **🌈 语义高亮**：自动识别分析报告中的**人名（蓝）**、**标签（绿）**、**重点（金）**，视觉层次分明。
- **🧠 深度回溯**：内置稳健的消息抓取器，支持熔断与跳跃机制，精准获取目标用户的历史发言。
- **🛡️ 智能抗扰**：优化的发送逻辑，图片直发，彻底解决客户端自动“@”的烦恼。
- **⚙️ 独立模型**：支持为画像功能单独指定 LLM 模型。

## 📦 安装与依赖

由于使用了浏览器渲染技术，本插件需要额外的依赖库。

1. **安装两个 Python 库（可跳过，已写在requirements.txt自动安装）**：
   ```bash
   pip install markdown playwright
   ```

2. **安装浏览器内核**（只需执行一次）：
   ```bash
   playwright install chromium
   ```

## ⚙️ 配置说明

请在 AstrBot 管理面板 -> 插件 -> `astrbot_plugin_portrayal_core` -> 配置 中进行设置。

| 配置项 | 类型 | 说明 | 默认值 |
| :--- | :--- | :--- | :--- |
| `llm_provider_id` | String | **[必填]** 指定用于分析的模型 ID (如 `openai_provider` 或 `gemini_provider`)，留空则尝试使用当前会话模型。 | `""` |
| `enable_image_output` | Bool | **[建议开启]** 是否生成高清图片卡片。关闭则发送纯文本。 | `true` |
| `batch_size` | Int | 单次 API 拉取的消息数量（建议 100）。 | `100` |
| `max_query_rounds` | Int | 默认回溯轮数。总条数 ≈ 轮数 * batch_size。 | `20` |
| `system_prompt_template` | Text | 自定义侧写师的人设与分析维度提示词。 | (内置默认) |

## ⌨️ 使用指令

**基本语法：**
`/画像 [可选: @某人] [可选: 回溯轮数] [可选: 更新/刷新]`

| 指令示例 | 说明 |
| :--- | :--- |
| `/画像` | 分析**发送者自己**的性格画像（使用缓存或默认轮数）。 |
| `/画像 @张三` | 分析群友 **@张三** 的性格画像。 |
| `/画像 50` | 分析**自己**，强制回溯最近 **50轮** (约5000条) 历史记录。 |
| `/画像 @李四 更新` | 强制忽略缓存，重新拉取 **@李四** 的最新消息进行分析。 |

## 🛠️ 常见问题

**Q: 为什么提示 "生成失败：LLM 拒绝生成内容"？**
A: 这通常是因为聊天记录中包含了敏感词（脏话、政治、色情等），触发了模型（特别是 Gemini）的安全过滤机制。建议尝试更换安全审查较宽松的模型，或者减少回溯轮数。

**Q: 图片发送失败，或者提示缺少依赖？**
A: 请确保你已经运行了 `playwright install chromium`。如果服务器环境（如 Docker）无法运行浏览器，插件会自动降级为发送纯文本。

**Q: 为什么生成的图片里没有引用原文？**
A: 插件会尝试引用你的指令消息。如果获取不到消息 ID（例如在某些特殊的适配器环境下），为了保证消息能发出去，插件会选择不引用直接发送。
