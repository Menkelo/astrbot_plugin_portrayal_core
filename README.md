# AstrBot Plugin Portrayal Core

_✨ 深度群友性格侧写 & 高清画像生成 (Core) ✨_

## 💡 简介

**人物画像(Core)** 是一款基于 AstrBot 的高级性格分析插件。

它不只是简单总结，而是通过深度回溯 OneBot 历史消息，结合 **MBTI + 大五人格 + 九型人格** 三大成熟体系，利用大语言模型（LLM）做心理学维度的侧写分析，并最终渲染为一张**极简杂志风（森绿主题）的高清侧写卡片**。

**🔥 核心特性：**
- **🎨 杂志风渲染**：基于 Playwright 无头浏览器渲染，生成「极简杂志 · 森绿主题」高清卡片——刊头条、悬浮头像、昵称大标题、小节自动编号。
- **🧠 三大人格体系**：内置基于 **MBTI**（四字母类型）、**大五人格 Big Five**（OCEAN 雷达）、**九型人格 Enneagram**（核心动机）的侧写提示词，结论专业、维度清晰，且要求证据优先、拒绝 AI 味。
- **🌈 语义排版**：自动加粗人名、`【标签】` 转绿色标签药丸、`##` 小节自动生成 `01 / 02` 编号、`>` 引用渲染为金句卡片。
- **🧩 深度回溯**：内置稳健的消息抓取器，支持熔断与跳跃机制，精准获取目标用户的历史发言。
- **🛡️ 智能抗扰**：图片直发 + 引用回复，彻底解决客户端自动「@」的烦恼。
- **⚙️ 独立模型 & 优雅降级**：可为画像单独指定 LLM 模型；缺少渲染依赖时自动降级为纯文本输出，**不再导致插件加载失败**。
- **🙂 头像兜底**：抓不到 QQ 头像时，自动用昵称首字母 + 哈希配色生成字母头像。

## 📦 安装与依赖

由于使用了浏览器渲染技术，本插件需要额外的依赖库。

1. **安装 Python 库（可跳过，已写在 requirements.txt 自动安装）**：
   ```bash
   pip install markdown playwright
   ```

2. **安装浏览器内核**（只需执行一次）：
   ```bash
   playwright install chromium
   ```

> 若服务器环境（如精简版 Docker）无法运行浏览器，可不安装上述依赖，插件会自动降级为纯文本输出。

## ⚙️ 配置说明

请在 AstrBot 管理面板 -> 插件 -> `astrbot_plugin_portrayal_core` -> 配置 中进行设置。

| 配置项 | 类型 | 说明 | 默认值 |
| :--- | :--- | :--- | :--- |
| `llm_provider_id` | String | 指定用于分析的模型 ID（如 `openai_provider`），留空则使用当前会话默认模型。 | `""` |
| `enable_image_output` | Bool | **[建议开启]** 是否生成高清图片卡片。关闭则发送纯文本。 | `true` |
| `max_fetch_duration` | Int | 默认消息回溯限时（秒）。时间越长回溯越深，但等待越久。 | `20` |
| `system_prompt_template` | Text | 侧写师人设与分析框架，默认基于 MBTI + 大五人格 + 九型人格。支持占位符 `{nickname}`、`{gender}`。 | (内置默认) |

## 📐 报告与排版约定

若你要自定义 `system_prompt_template`，让模型遵循以下约定即可获得最佳渲染效果：

| 写法 | 渲染效果 |
| :--- | :--- |
| `## 小节标题` | 自动生成 `01 / 02` 序号小节（标题内**不要**再手写数字） |
| `【人格标签】` | 森绿色标签药丸，如 `【INTP·逻辑学家】` |
| `**重点结论**` | 加粗 + 下划线高亮 |
| `> 金句` | 引用块，适合放结尾的「一句话神评」 |
| 昵称本身 | 全文出现的昵称会被**自动加粗** |

> **关于小节序号**：序号由渲染端的 CSS 计数器自动生成，因此**只对小节标题生效**。为最大化容错，插件会在渲染前归一化标题：
> - 整行纯加粗（如 `**核心动机**`，即使没写 `##`）也会被识别为小节标题；
> - 标题里自带的人工编号（`一、` `1.` `（1）` 等）会被自动剥离，避免与自动序号叠加导致「01 1. 标题」这类重复。
>
> 即便如此，仍建议在提示词中要求模型统一用 `## 标题` 且不要自己写编号，效果最稳。

占位符：`{nickname}` 为目标昵称，`{gender}` 为「他 / 她 / TA」。

## ⌨️ 使用指令

**基本语法：**
`/画像 [可选: @某人] [可选: 回溯秒数] [可选: 更新/刷新]`

| 指令示例 | 说明 |
| :--- | :--- |
| `/画像` | 分析**发送者自己**的性格画像（使用缓存或默认时长）。 |
| `/画像 @张三` | 分析群友 **@张三** 的性格画像。 |
| `/画像 50` | 分析**自己**，强制回溯最近 **50 秒**内的群消息。 |
| `/画像 @李四 更新` | 强制忽略缓存，重新拉取 **@李四** 的最新消息进行分析。 |

## 🛠️ 常见问题

**Q: 为什么提示 "生成失败：LLM 拒绝生成内容"？**
A: 通常是聊天记录中包含敏感词（脏话、政治、色情等），触发了模型（特别是 Gemini）的安全过滤。建议更换审查较宽松的模型，或减少回溯时长。

**Q: 图片发送失败，或提示缺少依赖？**
A: 请确认已运行 `playwright install chromium`。若服务器无法运行浏览器，插件会自动降级为纯文本。

**Q: 为什么生成的图片里没有引用原文？**
A: 插件会尝试引用你的指令消息，优先读取 SDK 规范字段 `message_obj.message_id`，并依次回退到原始消息字典与 `raw_data`。若全部获取不到消息 ID（某些特殊适配器环境），为保证消息发出，会改为不引用直接发送，同时在日志打印 `未获取到触发消息ID` 告警，便于排查。

**Q: 卡片里的小节序号有时缺失或错乱？**
A: 序号由 CSS 计数器自动生成，只对小节标题生效。新版已在渲染前归一化标题（纯加粗标题也识别、自动剥离人工编号），多数情况可自愈。若仍异常，多半是模型把小节写成了普通段落或列表项——请确保提示词要求统一使用 `## 标题`。

**Q: 卡片上的头像是个字母圆圈？**
A: 表示未能从 QQ 接口抓到该用户头像（接口偶发 403 或网络受限），此时用昵称首字母头像兜底，不影响其余内容。
